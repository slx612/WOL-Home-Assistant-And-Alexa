# PC Power Free

Language:

- [English](README.md)
- [Español](docs/README.es.md)

Local, subscription-free power control for a Windows PC through Home Assistant and Alexa.

Current status: `feature-complete, pending real-world testing`.

## What this is

This project covers the full local flow:

- Install a normal Windows program
- Let Home Assistant discover the PC automatically
- Pair it with a temporary code
- Wake it with Wake-on-LAN
- Shut it down or restart it over the local network
- Expose it to Alexa through Home Assistant without paying for a third-party subscription

No cloud dependency. No open internet ports required.

## How it works

### Power on

1. Home Assistant sends a Wake-on-LAN magic packet.
2. The PC powers on if BIOS/UEFI and Windows are configured correctly.

### Shutdown and restart

1. Home Assistant calls the local Windows agent.
2. The agent validates the source network and internal token.
3. The agent runs `shutdown` or `restart`.

### Discovery and pairing

1. The Windows agent advertises the PC over `zeroconf` on the local network.
2. Home Assistant discovers it automatically.
3. The user enters a temporary 6-digit pairing code generated by the Windows setup program.
4. Home Assistant exchanges that code for the internal token and stores the configuration.

## Requirements

- Home Assistant on the same local network
- A Windows PC with Wake-on-LAN support
- BIOS/UEFI with Wake-on-LAN available
- Alexa is optional and works through Home Assistant

## Windows installation

Recommended path:

1. Run [`windows_agent/dist/pcpowerfree-windows-x64-setup.exe`](windows_agent/dist/pcpowerfree-windows-x64-setup.exe)
2. Complete the installer
3. Review the detected settings
4. Leave these enabled:
   - `Create firewall rule`
   - `Install at startup`
5. Click `Install`
6. Save the pairing code

The setup program auto-detects:

- Computer name
- Active network adapter
- Current IP
- Primary MAC
- Wake-on-LAN broadcast
- Discovery subnet

The setup UI now supports `English` and `Spanish`.

## Home Assistant installation

### Manual

1. Copy `custom_components/pc_power_free` into `/config/custom_components/`
2. Restart Home Assistant
3. Go to `Settings > Devices & services`
4. Add `PC Power Free`

### HACS

The repository is prepared for HACS with [`hacs.json`](hacs.json).

1. In HACS, open `Custom repositories`
2. Add the URL of this repository
3. Type: `Integration`
4. Install `PC Power Free`
5. Restart Home Assistant

For the `default HACS list`, there is an extra maintainer process after the
repository is already working as a custom repository:

1. Make the GitHub Actions pass without ignored checks
2. Publish a full GitHub release
3. Open a PR against `hacs/default`

Checklist: [`docs/HACS_PUBLISHING.md`](docs/HACS_PUBLISHING.md)

## Pairing with Home Assistant

### Recommended flow

1. Install the Windows program
2. Install the Home Assistant integration
3. Wait for the PC to appear automatically
4. Select the discovered PC
5. Enter the temporary pairing code shown by the Windows setup program
6. Confirm the device name

### If automatic discovery fails

There is also a manual IP flow:

1. `Add integration`
2. `PC Power Free`
3. `Set up by IP manually`
4. Enter the PC IP and agent port
5. Enter the pairing code

## Alexa

The supported path is `Alexa + Home Assistant`.

The simplest local route is `emulated_hue`.

Example:

```yaml
emulated_hue:
  listen_port: 80
  entities:
    switch.pc_despacho_power:
      name: "Office PC"
      hidden: false
```

## Security

- Do not expose port `8777` to the internet
- Restrict access to the Home Assistant IP or at least your LAN
- Use a VPN if you need remote access
- The pairing code is temporary
