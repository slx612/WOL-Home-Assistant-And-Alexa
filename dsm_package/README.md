# DSM Package

This directory now contains an initial Synology DSM package scaffold for `PC Power Free`.

Current intent:

- reuse the shared runtime from `agent_core`
- reuse the Linux runtime from `linux_agent`
- add DSM-specific packaging in `.spk` format
- stay conservative around DSM 7 privilege rules

What is included here:

- package metadata template in `template/INFO.in`
- DSM 7 privilege configuration in `template/conf/privilege`
- package lifecycle scripts in `template/scripts`
- DSM runtime wrappers in `payload/dsm_runtime`
- local build script in `build-dsm-package.ps1`

Important constraints:

- DSM package versions must be numeric in `INFO`, so the builder maps the app version `0.2.0-beta.6` to the DSM package version `0.2.0-0006`
- the package still assumes a usable `python3` is available on the NAS
- the package now vendors the pure-Python `ifaddr` and `zeroconf` dependencies into `app/vendor`, so it no longer depends on those modules already being installed on the NAS
- the package reuses the Linux agent logic and will generate its config under the package directory
- this scaffold does not yet claim SynoCommunity-ready status
- shutdown and restart support must still be validated against DSM 7 privilege expectations before they should be advertised as safe NAS power controls
- real DSM 7.2 validation now covers install, service start, discovery, and pairing on hardware; power actions are still pending validation

Current package layout generated by the builder:

- `INFO`
- `package.tgz`
- `scripts/`
- `conf/privilege`
- `PACKAGE_ICON.PNG`
- `PACKAGE_ICON_256.PNG`
- `LICENSE`

How to build locally on Windows:

```powershell
.\dsm_package\build-dsm-package.ps1
```

That produces:

- `dsm_package\build\package.tgz`
- `dsm_package\dist\pcpowerfree-dsm-noarch-<version>.spk`

What still needs to happen before calling this publishable:

- verification of the Python runtime strategy on DSM
- a DSM-facing setup experience better than reading package logs and summary files
- privilege review for shutdown and restart actions
- SynoCommunity packaging polish if this moves into `spksrc`
